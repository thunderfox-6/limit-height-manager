# 限高架智能管家

> 铁路桥涵限高防护架数字台账与预警系统

## 项目简介

限高架智能管家是一套基于Web的限高架数字管理系统，用于铁路工务段对管内的桥梁、涵洞限高防护架进行数字化管理。

### 核心功能

- 📊 **电子台账管理**：建立限高架电子台账，支持Excel批量导入
- ✅ **合规自动校验**：依据TG/GW 212-2017规范自动校验，红黄绿蓝四色标识
- 🔃 **上下行分离管理**：公路侧/铁路侧分开管理，独立展示和校验
- 📹 **视频监控集成**：支持关联视频监控URL，在详情页播放
- 🤖 **智能推荐**：根据桥涵净高智能推荐建议的限高架净高
- ⚠️ **预警管理**：自动生成预警记录，支持闭环处理

### 合规校验规则

| 状态 | 颜色 | 判定条件 | 说明 |
|------|------|----------|------|
| 严重隐患 | 🔴 红色 | 限高架净高 ≥ 桥涵净高 | 存在碰撞风险 |
| 余量不足 | 🟡 黄色 | 余量 < 0.02米 | 安全余量不足 |
| 符合规范 | 🟢 绿色 | 0.02米 ≤ 余量 ≤ 0.10米 | 符合TG/GW 212-2017 |
| 限高过严 | 🔵 蓝色 | 余量 > 0.10米 | 可考虑提升限高值 |

## 技术栈

- **前端框架**: Next.js 16 + React 19 + TypeScript
- **UI组件**: Tailwind CSS + shadcn/ui
- **数据库**: Prisma + SQLite
- **Excel处理**: xlsx

## 快速开始

### 环境要求

- Node.js 18+
- npm / yarn / bun

### 安装依赖

```bash
npm install
# 或
yarn install
# 或
bun install
```

### 初始化数据库

```bash
npm run db:push
```

### 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000 即可使用系统。

### 默认账号

- 用户名: `admin`
- 密码: `123456`

## Excel导入模板

系统支持Excel批量导入，模板字段如下：

| 字段名 | 说明 | 必填 |
|--------|------|------|
| 桥涵名称 | 桥涵的名称 | ✅ |
| 里程 | 如 K12+345 | ✅ |
| 桥涵净高 | 单位：米 | ✅ |
| 下行净高 | 单位：米 | ✅ |
| 上行净高 | 单位：米，若无则留空 | ❌ |
| 道路等级 | 国道/省道/县道/城市道路/乡村道路 | ✅ |
| 所属工区 | 如"石家庄车间" | ✅ |
| 结构形式 | 门式/三角架式/桁架式 | ❌ |
| 安装日期 | YYYY-MM-DD | ❌ |
| 有无传感器 | 0表示无，1表示有 | ❌ |
| 视频地址 | 视频URL或监控流地址 | ❌ |
| 测量日期 | YYYY-MM-DD | ❌ |

```bash

```

## 规范依据

本系统依据《TG/GW 212-2017 铁路桥涵限高防护架技术条件》进行合规性校验：

- 限高架净高应低于桥涵净高
- 安全余量建议为100mm（0.10米）
- 最小安全余量不应小于20mm（0.02米）
